# תרחישי שימוש (User Flows)

## צד העובד

### 1. התחברות
1. עובד מזין מספר עובד
2. המערכת מזהה ומציגה את שמו
3. כניסה למסך הראשי של העובד

### 2. הגשת זמינות
1. עובד רואה לוח שנה של הרבעון הנוכחי
2. על כל תאריך הוא יכול לסמן:
   - **לא יכול לעבוד** (אדום) — המערכת לא תציע את העובד ליום זה
   - **מעדיף לעבוד** (ירוק) — אינדיקציה חיובית למנהל
   - **מעדיף לא לעבוד** (כתום) — אינדיקציה שלילית למנהל, אבל לא חסימה
   - **ללא העדפה** (ברירת מחדל) — רגיל
3. העובד שומר ויכול לערוך עד למועד שנקבע

## צד המנהל

### 1. התחברות
1. מנהל מזין מספר עובד + סיסמה
2. כניסה למסך הניהול הראשי

### 2. העלאת רשימת תאריכי משמרות (תחילת רבעון)
1. מנהל לוחץ "העלאת רשימת תאריכים"
2. מצרף קובץ
3. המערכת מנתחת את הקובץ ומציגה תצוגה מקדימה:
   - רשימת תאריכים
   - סוג משמרת לכל תאריך
   - סימון סופ"ש (חמישי-שבת) אוטומטי
4. מנהל מאשר → הנתונים נשמרים ב-DB

### 3. מסך שיבוץ ראשי
**פריסת המסך:**

```
┌─────────────────────────────────────┬──────────────────────────┐
│                                     │                          │
│        רשימת תאריכי משמרות   │               עובדים זמינים  │
│    (לפי סדר כרונולוגי)             │   (מסוננים לפי משמרת   │
│                                     │    הנוכחית שנבחרה)      │
│    ┌─────────────────────────┐      │                          │
│    │ 15/01/2026 - קצין       │◄─────│   ● עובד א' (OL1)       │
│    │ [שם עובד משובץ]        │      │     🟢 מעדיף לעבוד     │
│    │ או [לא משובץ]           │      |   ⏱ 45 ימים מאז משמרת │
│    └─────────────────────────┘      │                          │
│                                     │   ● עובד ב' (OL2)       │
│    ┌─────────────────────────┐      │     🟡 מעדיף לא          │
│    │ 22/01/2026 - שמירה     │      │     ⏱ 90 ימים מאז משמרת │
│    │ [לא משובץ]              │      │                          │
```

### 4. שיבוץ עובד למשמרת (Drag & Drop)

1. מנהל לוחץ על תאריך משמרת ברשימה (או בוחר אותו)
2. רשימת העובדים בצד ימין מתעדכנת ומציגה:
   - **כשירים וזמינים** — מוצגים ראשונים, ממוינים לפי ימים מאז משמרת אחרונה (הכי הרבה זמן = ראשון)
   - **כשירים אבל מעדיפים לא** — מוצגים עם אינדיקציה כתומה
   - **לא כשירים** — מוצגים עם אינדיקציה אדומה
   - **לא זמינים (הגישו שלא יכולים)** — מוצגים עם חסימה
   - **פטורים** — מוצגים עם סימון פטור
   - **כבר שובצו הרבעון** — מוצגים עם סימון ✓
3. מנהל גורר עובד לתאריך משמרת

#### אינדיקציות בעת שיבוץ

| מצב | אינדיקציה |
|---|---|
| עובד כשיר וזמין | ירוק — שיבוץ רגיל |
| עובד מעדיף לא לעבוד ביום זה | כתום — אזהרה קלה |
| עובד לא כשיר לסוג משמרת זה | **אדום גדול** — אזהרה חמורה, שיבוץ עדיין אפשרי |
| עובד הגיש שלא יכול ביום זה | **אדום גדול** — אזהרה חמורה, שיבוץ עדיין אפשרי |
| עובד כבר עשה סופ"ש השנה | **אדום גדול** — חריגה מכלל שנתי |
| עובד כבר שובץ הרבעון | אזהרה — אבל אפשרי (החלפות, חריגים) |
| עובד פטור | **אדום גדול** — שיבוץ עדיין אפשרי |

> **עיקרון מנחה**: המערכת לעולם לא חוסמת שיבוץ. היא רק מציגה אזהרות ברמות שונות. למנהל יש את המילה האחרונה.

### 5. שיבוץ כפוי

כשהמנהל משבץ עובד למרות אזהרה אדומה:
1. מודאל קופץ: "שיבוץ זה חורג מהכללים. סיבה?"
2. מנהל מזין סיבה בטקסט חופשי
3. השיבוץ נשמר עם `is_forced = true` והסיבה

### 6. שמירת התקדמות

1. מנהל לוחץ "ייצוא התקדמות"
2. המערכת מייצאת קובץ JSON עם כל מצב הרבעון:
   - תאריכי משמרות
   - שיבוצים שבוצעו
   - זמינות עובדים
3. לחצן "ייבוא התקדמות" — מנהל מעלה JSON וממשיך מאותה נקודה

### 7. סקירת סיכום

לפני פרסום סופי, מסך סיכום מציג:
- כל השיבוצים
- עובדים שטרם שובצו
- שיבוצים כפויים (מסומנים)
- חריגות מכללים
